# L90 — Persona-Action Legacy-Stub Remediation v1

## Purpose

Define the deterministic truth conditions for the first bounded `P2` wave that replaces selected legacy-stub aliases with real handlers.

## Bounded Alias Set

- `task.create`
- `follow-up.schedule`
- `reminder.send`

## Claim

For the bounded alias set above, `P2` may proceed only if:

1. every alias currently classified `legacy-stubbed` has an explicit target replacement path
2. the replacement path is deterministic and test-bound
3. the phase keeps any still-unsupported aliases outside the bounded write set

## Expected Outcome

The first bounded `P2` wave may convert the three targeted aliases from literal-only legacy stubs into truthful mapped handlers or fail closed before code extraction.
