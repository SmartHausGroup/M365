# SMARTHAUS M365 Pilot and Rollout Model v2

## Purpose

Define rollout phases, acceptance gates, and expansion criteria for the expanded workforce product.

## Authority

- **Contract:** `registry/pilot_rollout_model_v2.yaml`

## Rollout Phases

1. **Pilot Scope** — single tenant, core tier, operations department.
2. **Pilot Acceptance** — governed action success, audit completeness, sign-off.
3. **Expansion Criteria** — pilot acceptance required before broader rollout.
4. **Rollout Waves** — Wave 1 (core/ops), Wave 2 (core/all active), Wave 3 (expansion).
