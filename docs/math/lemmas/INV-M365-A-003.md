# Lemma: Syntactic validity and disjoint action sets

-----------------------------------------
**Lemma ID:** LEM-M365-A-003-01

**Invariant:** INV-M365-A-003

**Derived from:**
- Section 2: Instruction Structure (action–persona compatibility, valid instruction)

**Statement:**

For every instruction \(x = (p, i, a, \mathit{params})\) that is accepted for evaluation (i.e., for which \(\mathrm{Eval}(x, \tau)\) is defined for some \(\tau\)), \(x\) is syntactically valid: \((p = \mathsf{Admin} \wedge a \in \mathcal{A}_{\mathsf{Admin}}) \vee (p = \mathsf{User} \wedge a \in \mathcal{A}_{\mathsf{User}})\). Furthermore \(\mathcal{A}_{\mathsf{Admin}} \cap \mathcal{A}_{\mathsf{User}} = \varnothing\).

**Assumptions:**

- Phase 1 defines \(\mathcal{A}_{\mathsf{Admin}} \subseteq \mathcal{A}\), \(\mathcal{A}_{\mathsf{User}} \subseteq \mathcal{A}\), and the constraint \(\mathcal{A}_{\mathsf{Admin}} \cap \mathcal{A}_{\mathsf{User}} = \varnothing\) (Section 2).
- \(x\) is syntactically valid iff \((p = \mathsf{Admin} \wedge a \in \mathcal{A}_{\mathsf{Admin}}) \vee (p = \mathsf{User} \wedge a \in \mathcal{A}_{\mathsf{User}})\); \(\mathcal{X}_{\mathrm{valid}}\) denotes the set of syntactically valid instructions (Section 2).
- Section 6 states that if \(x \notin \mathcal{X}_{\mathrm{valid}}\), then \(\mathrm{Eval}(x, \tau)\) is undefined. So whenever \(\mathrm{Eval}(x, \tau)\) is defined, \(x \in \mathcal{X}_{\mathrm{valid}}\) (contrapositive).

**Proof outline:**

1. **Disjointness:** By Section 2, \(\mathcal{A}_{\mathsf{Admin}} \cap \mathcal{A}_{\mathsf{User}} = \varnothing\) is an explicit constraint. Hence the Admin and User action sets have no common element.

2. **Accepted implies valid:** Suppose \(x\) is accepted for evaluation, i.e. there exists \(\tau \in \mathcal{T}\) such that \(\mathrm{Eval}(x, \tau)\) is defined. Section 6 states that if \(x \notin \mathcal{X}_{\mathrm{valid}}\), then \(\mathrm{Eval}(x, \tau)\) is undefined. By contraposition, if \(\mathrm{Eval}(x, \tau)\) is defined then \(x \in \mathcal{X}_{\mathrm{valid}}\). So \(x\) satisfies the syntactic validity condition: \((p = \mathsf{Admin} \wedge a \in \mathcal{A}_{\mathsf{Admin}}) \vee (p = \mathsf{User} \wedge a \in \mathcal{A}_{\mathsf{User}})\).

3. **Case analysis:** For \(x \in \mathcal{X}_{\mathrm{valid}}\), either (a) \(p = \mathsf{Admin}\) and \(a \in \mathcal{A}_{\mathsf{Admin}}\), or (b) \(p = \mathsf{User}\) and \(a \in \mathcal{A}_{\mathsf{User}}\). In (a), \(a \notin \mathcal{A}_{\mathsf{User}}\) by disjointness; in (b), \(a \notin \mathcal{A}_{\mathsf{Admin}}\) by disjointness. So persona and action are consistent with exactly one of the two sets.

**Edge cases:**

- If \(p = \mathsf{Admin}\) but \(a \notin \mathcal{A}_{\mathsf{Admin}}\), then \(x \notin \mathcal{X}_{\mathrm{valid}}\), so \(\mathrm{Eval}(x, \tau)\) is undefined—such an instruction is not "accepted for evaluation" in the sense of producing an outcome; the invariant applies to accepted instructions only.
- If an action were in both sets, disjointness would be violated; the spec forbids that.

**Conclusion:**

Phase 1 constrains undefinedness of \(\mathrm{Eval}\) to include \(x \notin \mathcal{X}_{\mathrm{valid}}\), and defines \(\mathcal{X}_{\mathrm{valid}}\) and disjointness of action sets. Therefore every instruction accepted for evaluation is syntactically valid and the Admin/User action sets are disjoint. Hence INV-M365-A-003 holds.
