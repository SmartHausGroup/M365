# Lemma: Exactly one audit record per successful Admin mutation

-----------------------------------------
**Lemma ID:** LEM-M365-E-001-01

**Invariant:** INV-M365-E-001

**Derived from:**
- Section 5: Audit Event Semantics (successful Admin mutation)

**Statement:**

For all \(x = (\mathsf{Admin}, i, a, \mathit{params})\) with \(a \in \mathcal{A}_{\mathrm{mut}}\), and for all \(\tau \in \mathcal{T}\) for which \(\mathrm{Eval}(x, \tau)\) is defined and the result \(r\) indicates success: if \(\mathrm{Eval}(x, \tau) = (r, \tau', \mathit{audit})\), then \(|\mathit{audit}| = 1\).

**Assumptions:**

- Section 5: "If \(x = (\mathsf{Admin}, i, a, \mathit{params})\), \(a \in \mathcal{A}_{\mathrm{mut}}\), and \(\mathrm{Eval}(x, \tau) = (r, \tau', \mathit{audit})\) with \(r\) indicating success, then \(|\mathit{audit}| = 1\)."
- Phase 1 does not define the structure of \(\mathcal{R}\); "success" is a condition on \(r\) that the spec refers to explicitly for this rule.

**Proof outline:**

1. Let \(x = (\mathsf{Admin}, i, a, \mathit{params})\) with \(a \in \mathcal{A}_{\mathrm{mut}}\), \(\tau \in \mathcal{T}\), and suppose \(\mathrm{Eval}(x, \tau)\) is defined with result \(r\) indicating success.
2. Then \(\mathrm{Eval}(x, \tau) = (r, \tau', \mathit{audit})\) for unique \((r, \tau', \mathit{audit})\) (determinism).
3. By Section 5, under these hypotheses—\(x\) has Admin persona, \(a \in \mathcal{A}_{\mathrm{mut}}\), and \(r\) indicates success—the specification states \(|\mathit{audit}| = 1\).
4. Therefore the audit sequence has exactly one element.

**Edge cases:**

- If \(r\) does not indicate success (e.g. failure), Section 5 instead requires \(\mathit{audit} = ()\) (no audit on failure); this lemma applies only when \(r\) indicates success.
- If \(a \notin \mathcal{A}_{\mathrm{mut}}\) or persona is not Admin, the "successful Admin mutation" condition does not hold; audit cardinality is constrained by other rules (Section 4, Section 5 read-only/User).

**Conclusion:**

The Phase 1 rule for successful Admin mutation explicitly requires exactly one audit record in that case. Hence INV-M365-E-001 holds.
