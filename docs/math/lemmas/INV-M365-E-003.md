# Lemma: Zero audit records for read-only and User-persona actions

-----------------------------------------
**Lemma ID:** LEM-M365-E-003-01

**Invariant:** INV-M365-E-003

**Derived from:**
- Section 4: Mutation vs Read Semantics (read-only)
- Section 5: Audit Event Semantics (read-only and User)

**Statement:**

For every instruction \(x = (p, i, a, \mathit{params})\) and \(\tau \in \mathcal{T}\) for which \(\mathrm{Eval}(x, \tau)\) is defined: if \(p = \mathsf{User}\) or \(a \in \mathcal{A}_{\mathrm{read}}\), then \(\mathrm{Eval}(x, \tau) = (r, \tau', \mathit{audit})\) implies \(|\mathit{audit}| = 0\).

**Assumptions:**

- Section 4: for \(a \in \mathcal{A}_{\mathrm{read}}\), when \(\mathrm{Eval}(x, \tau)\) is defined, \(\mathrm{Eval}(x, \tau) = (r, \tau', \mathit{audit}) \Rightarrow \tau' = \tau \wedge \mathit{audit} = ()\).
- Section 5: "For \(a \in \mathcal{A}_{\mathrm{read}}\), or for User persona, we already required \(\mathit{audit} = ()\) when \(\mathrm{Eval}\) is defined (Section 4). So audit events are only for successful Admin mutations."

**Proof outline:**

1. **Read-only actions.** Let \(a \in \mathcal{A}_{\mathrm{read}}\). By Section 4, whenever \(\mathrm{Eval}(x, \tau)\) is defined and \(\mathrm{Eval}(x, \tau) = (r, \tau', \mathit{audit})\), we have \(\mathit{audit} = ()\). So \(|\mathit{audit}| = 0\).

2. **User persona.** Let \(p = \mathsf{User}\). By Section 4, \(\mathcal{A}_{\mathrm{mut}} \subseteq \mathcal{A}_{\mathsf{Admin}}\), so User cannot have \(a \in \mathcal{A}_{\mathrm{mut}}\) (from persona-action compatibility and disjointness). So for User, either \(a \in \mathcal{A}_{\mathrm{read}}\) or \(a\) is in some other subset of \(\mathcal{A}_{\mathsf{User}}\) that is not \(\mathcal{A}_{\mathrm{mut}}\). Section 5 states that for User persona we already required \(\mathit{audit} = ()\) when \(\mathrm{Eval}\) is defined—i.e. audit events are only for successful Admin mutations, so non-Admin or non-mutating cases produce no audit. Thus when \(p = \mathsf{User}\), \(\mathit{audit} = ()\) when \(\mathrm{Eval}\) is defined, so \(|\mathit{audit}| = 0\).

3. **Combined.** For every evaluation that is defined, if persona is User or action is read-only, the specification requires \(\mathit{audit} = ()\), hence \(|\mathit{audit}| = 0\).

**Edge cases:**

- If \(p = \mathsf{Admin}\) and \(a \in \mathcal{A}_{\mathrm{mut}}\) and \(r\) indicates success, Section 5 requires \(|\mathit{audit}| = 1\); that case is excluded by the condition "persona = User or action ∈ A_read".
- If \(\mathrm{Eval}\) is undefined, there is no output; the invariant concerns defined evaluations only.

**Conclusion:**

Section 4 and Section 5 require that when \(\mathrm{Eval}\) is defined, read-only actions and User persona yield \(\mathit{audit} = ()\). Hence for every defined evaluation with persona User or action in \(\mathcal{A}_{\mathrm{read}}\), \(|\mathit{audit}| = 0\). Therefore INV-M365-E-003 holds.
