# Lemma: Determinism of evaluation (keystone)

-----------------------------------------
**Lemma ID:** LEM-M365-C-001-01

**Invariant:** INV-M365-C-001

**Derived from:**
- Section 3: Core Evaluation Function(s)
- Section 6: Determinism and Failure Constraints

**Statement:**

For all \(x \in \mathcal{X}\) and \(\tau \in \mathcal{T}\), there is at most one \((r, \tau', \mathit{audit})\) such that \(\mathrm{Eval}(x, \tau) = (r, \tau', \mathit{audit})\). Equivalently: for the same instruction and the same tenant state, evaluation produces exactly one outcome—either a unique triple \((r, \tau', \mathit{audit})\) when defined, or undefined (rejection) with no side effects; no two evaluations with the same \((x, \tau)\) may produce different outputs.

**Assumptions:**

- \(\mathrm{Eval}: \mathcal{X} \times \mathcal{T} \rightharpoonup \mathcal{R} \times \mathcal{T} \times \mathcal{E}^*\) is a **partial function** (Section 3). By definition, a partial function assigns at most one value to each argument pair.
- Section 3 explicitly states: for the same \(x\) and \(\tau\), \(\mathrm{Eval}(x, \tau)\) is either undefined for both, or defined with the same \((r, \tau', \mathit{audit})\); and \(\mathrm{Eval}(x, \tau) = (r_1, \tau_1', \mathit{audit}_1) \wedge \mathrm{Eval}(x, \tau) = (r_2, \tau_2', \mathit{audit}_2) \Rightarrow r_1 = r_2 \wedge \tau_1' = \tau_2' \wedge \mathit{audit}_1 = \mathit{audit}_2\).
- Section 6 states: \(\forall x \in \mathcal{X},\, \forall \tau \in \mathcal{T}\), at most one \((r, \tau', \mathit{audit})\) such that \(\mathrm{Eval}(x, \tau) = (r, \tau', \mathit{audit})\).

**Proof outline:**

1. **Partial function:** The signature \(\mathrm{Eval}: \mathcal{X} \times \mathcal{T} \rightharpoonup \mathcal{R} \times \mathcal{T} \times \mathcal{E}^*\) denotes a partial function. By the mathematical definition of a partial function, for each \((x, \tau) \in \mathcal{X} \times \mathcal{T}\), \(\mathrm{Eval}\) maps \((x, \tau)\) to at most one element of \(\mathcal{R} \times \mathcal{T} \times \mathcal{E}^*\) (or to nothing). Hence uniqueness of outcome when defined is inherent.

2. **When defined:** If \(\mathrm{Eval}(x, \tau)\) is defined, let \((r, \tau', \mathit{audit})\) be its value. Any other evaluation of \((x, \tau)\) must yield the same value, otherwise the map would not be a function. Section 3 makes this explicit: two equal images imply \(r_1 = r_2\), \(\tau_1' = \tau_2'\), \(\mathit{audit}_1 = \mathit{audit}_2\).

3. **When undefined:** If \(\mathrm{Eval}(x, \tau)\) is undefined, the instruction is rejected (Section 3). Section 6 requires that in that case there is no state change and no audit. So "rejection" is a unique outcome (undefined, no side effects); no other outcome may occur for the same \((x, \tau)\) because the partial function does not assign a value.

4. **No mixed outcomes:** It is impossible that one evaluation of \((x, \tau)\) is defined and another is undefined, because the value of \(\mathrm{Eval}(x, \tau)\) is fixed by the definition of the partial function—either it has a value or it does not, for the whole specification.

**Edge cases:**

- Non-determinism would require \(\mathrm{Eval}(x, \tau)\) to have more than one value for some \((x, \tau)\), which contradicts the partial function definition.
- "Same instruction and same tenant state" is exactly the domain pair \((x, \tau)\); the specification does not allow any other input to affect the result (Section 7).

**Conclusion:**

\(\mathrm{Eval}\) is defined in Phase 1 as a partial function on \(\mathcal{X} \times \mathcal{T}\), and Section 3 and Section 6 explicitly require deterministic outcome. Therefore for the same \((x, \tau)\), evaluation produces exactly one outcome (either a unique triple or undefined with no side effects). Hence INV-M365-C-001 holds. This lemma does not depend on any other invariant; it is the keystone from which other reasoning may assume determinism.
