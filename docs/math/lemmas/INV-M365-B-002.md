# Lemma: Only Admin may perform mutating actions

-----------------------------------------
**Lemma ID:** LEM-M365-B-002-01

**Invariant:** INV-M365-B-002

**Derived from:**
- Section 4: Mutation vs Read Semantics (persona and mutation)

**Statement:**

For every instruction \(x = (p, i, a, \mathit{params})\) with \(a \in \mathcal{A}_{\mathrm{mut}}\), \(p = \mathsf{Admin}\). Equivalently: for every instruction with \(p = \mathsf{User}\), \(a \notin \mathcal{A}_{\mathrm{mut}}\).

**Assumptions:**

- Phase 1 states: "Only Admin may perform mutating actions" and \(\mathcal{A}_{\mathrm{mut}} \subseteq \mathcal{A}_{\mathsf{Admin}}\) (Section 4).
- By Section 2, for accepted instructions, \((p = \mathsf{Admin} \wedge a \in \mathcal{A}_{\mathsf{Admin}}) \vee (p = \mathsf{User} \wedge a \in \mathcal{A}_{\mathsf{User}})\), and \(\mathcal{A}_{\mathsf{Admin}} \cap \mathcal{A}_{\mathsf{User}} = \varnothing\).

**Proof outline:**

1. By Section 4, \(\mathcal{A}_{\mathrm{mut}} \subseteq \mathcal{A}_{\mathsf{Admin}}\). So every mutating action is an Admin-allowed action.
2. For any instruction accepted for evaluation with \(a \in \mathcal{A}_{\mathrm{mut}}\), we have \(a \in \mathcal{A}_{\mathsf{Admin}}\) (from step 1).
3. By disjointness, \(a \notin \mathcal{A}_{\mathsf{User}}\). So the disjunct \(p = \mathsf{User} \wedge a \in \mathcal{A}_{\mathsf{User}}\) is false.
4. Syntactic validity requires \((p = \mathsf{Admin} \wedge a \in \mathcal{A}_{\mathsf{Admin}}) \vee (p = \mathsf{User} \wedge a \in \mathcal{A}_{\mathsf{User}})\). Since \(a \in \mathcal{A}_{\mathsf{Admin}}\) and \(a \notin \mathcal{A}_{\mathsf{User}}\), the second disjunct is false; hence \(p = \mathsf{Admin}\) must hold.
5. Therefore for every instruction with \(a \in \mathcal{A}_{\mathrm{mut}}\) that is accepted for evaluation, \(p = \mathsf{Admin}\). Contrapositively: if \(p = \mathsf{User}\), then \(a \notin \mathcal{A}_{\mathrm{mut}}\).

**Edge cases:**

- If \(p = \mathsf{User}\) and \(a \in \mathcal{A}_{\mathrm{mut}}\), then \(a \in \mathcal{A}_{\mathsf{Admin}}\) (since \(\mathcal{A}_{\mathrm{mut}} \subseteq \mathcal{A}_{\mathsf{Admin}}\)), so the instruction is not syntactically valid; \(\mathrm{Eval}\) is undefined. So no such instruction receives a defined evaluation outcome.

**Conclusion:**

The inclusion \(\mathcal{A}_{\mathrm{mut}} \subseteq \mathcal{A}_{\mathsf{Admin}}\) and the syntactic validity condition imply that whenever the action is mutating, the persona must be Admin. Hence INV-M365-B-002 holds.
