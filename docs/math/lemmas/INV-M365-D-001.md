# Lemma: Read-only actions preserve state and produce no audit

-----------------------------------------
**Lemma ID:** LEM-M365-D-001-01

**Invariant:** INV-M365-D-001

**Derived from:**
- Section 3: Core Evaluation Function(s) (read-only vs mutating)
- Section 4: Mutation vs Read Semantics (read-only)

**Statement:**

For all \(x = (p, i, a, \mathit{params})\) with \(a \in \mathcal{A}_{\mathrm{read}}\), and for all \(\tau \in \mathcal{T}\) for which \(\mathrm{Eval}(x, \tau)\) is defined: if \(\mathrm{Eval}(x, \tau) = (r, \tau', \mathit{audit})\), then \(\tau' = \tau\) and \(|\mathit{audit}| = 0\) (i.e. \(\mathit{audit}\) is the empty sequence).

**Assumptions:**

- \(\mathcal{A}_{\mathrm{read}}\) and \(\mathcal{A}_{\mathrm{mut}}\) partition the action set with \(\mathcal{A}_{\mathrm{read}} \cap \mathcal{A}_{\mathrm{mut}} = \varnothing\) (Section 3).
- Section 3: for read-only \(a \in \mathcal{A}_{\mathrm{read}}\), \(\mathrm{Eval}(x, \tau) = (r, \tau', \mathit{audit}) \Rightarrow \tau' = \tau \wedge \mathit{audit} = ()\).
- Section 4: for \(x = (p, i, a, \mathit{params})\) with \(a \in \mathcal{A}_{\mathrm{read}}\), when \(\mathrm{Eval}(x, \tau)\) is defined, \(\mathrm{Eval}(x, \tau) = (r, \tau', \mathit{audit}) \Rightarrow \tau' = \tau \wedge \mathit{audit} = ()\).

**Proof outline:**

1. Let \(x = (p, i, a, \mathit{params})\) with \(a \in \mathcal{A}_{\mathrm{read}}\), and \(\tau \in \mathcal{T}\) such that \(\mathrm{Eval}(x, \tau)\) is defined.
2. Then there exists a unique \((r, \tau', \mathit{audit})\) such that \(\mathrm{Eval}(x, \tau) = (r, \tau', \mathit{audit})\) (by determinism, Section 3 / Section 6).
3. By Section 3 (and Section 4), for read-only actions the specification requires: \(\mathrm{Eval}(x, \tau) = (r, \tau', \mathit{audit}) \Rightarrow \tau' = \tau \wedge \mathit{audit} = ()\).
4. Apply this implication: from step 2 we have \(\mathrm{Eval}(x, \tau) = (r, \tau', \mathit{audit})\), so \(\tau' = \tau\) and \(\mathit{audit} = ()\).
5. The empty sequence has length zero: \(|\mathit{audit}| = 0\).

**Edge cases:**

- If \(a \notin \mathcal{A}_{\mathrm{read}}\), this lemma does not apply; mutating actions may change state and produce audit (Section 4).
- If \(\mathrm{Eval}(x, \tau)\) is undefined, there is no output triple; the invariant constrains only defined evaluations for read-only actions.

**Conclusion:**

The Phase 1 requirement for read-only actions—that when \(\mathrm{Eval}\) is defined, \(\tau' = \tau\) and \(\mathit{audit} = ()\)—implies that for every instruction with action in \(\mathcal{A}_{\mathrm{read}}\) and defined evaluation, new_tenant_state equals tenant_state and the audit event list has length zero. Hence INV-M365-D-001 holds.
