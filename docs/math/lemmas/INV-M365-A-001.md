# Lemma: Instruction 4-tuple structure and completeness

-----------------------------------------
**Lemma ID:** LEM-M365-A-001-01

**Invariant:** INV-M365-A-001

**Derived from:**
- Section 1: Sets and Domains
- Section 2: Instruction Structure

**Statement:**

For all \(x\) that are inputs to evaluation, \(x \in \mathcal{X}\) where \(\mathcal{X} = \mathcal{P} \times \mathcal{I} \times \mathcal{A} \times \mathcal{P}\!\mathit{ar}\). Equivalently: every instruction has exactly four components—persona, identity, action, parameters—and each component is drawn from the corresponding set; no component may be omitted, and no component may be supplied by any source other than the instruction tuple itself.

**Assumptions:**

- Phase 1 defines \(\mathcal{X} = \mathcal{P} \times \mathcal{I} \times \mathcal{A} \times \mathcal{P}\!\mathit{ar}\) (Section 1).
- An instruction is \(x = (p, i, a, \mathit{params})\) with \(p \in \mathcal{P}\), \(i \in \mathcal{I}\), \(a \in \mathcal{A}\), \(\mathit{params} \in \mathcal{P}\!\mathit{ar}\) (Section 2).
- The domain of \(\mathrm{Eval}\) is \(\mathcal{X} \times \mathcal{T}\) (Section 3); therefore any input to evaluation is a pair whose first element is in \(\mathcal{X}\).

**Proof outline:**

1. By definition of Cartesian product, every element of \(\mathcal{X}\) is an ordered 4-tuple \((p, i, a, \mathit{params})\).
2. Hence every element of \(\mathcal{X}\) has exactly four components; no element of \(\mathcal{X}\) has fewer than four or more than four components.
3. For each position (first, second, third, fourth), the component is uniquely determined by the tuple; there is no alternate source for that position because \(\mathcal{X}\) is defined as the product set and the specification does not define any other input channel (Section 7: no hidden context).
4. Therefore any object accepted as an instruction for evaluation must be a 4-tuple with each component present and each component the sole source for that position.

**Edge cases:**

- If an object is not a 4-tuple, it is not in \(\mathcal{X}\), so it is not in the domain of \(\mathrm{Eval}\) as defined; the invariant fails (instruction structure violated).
- If any component were to be "filled in" from elsewhere, the specification would contradict Section 7 (instruction fully described by the tuple; no hidden context); the Phase 1 spec excludes that by non-goals.

**Conclusion:**

The Phase 1 definition of \(\mathcal{X}\) and the domain of \(\mathrm{Eval}\) implies that every instruction has exactly four components and that no component may be omitted or replaced by an alternate source. Hence INV-M365-A-001 holds.
