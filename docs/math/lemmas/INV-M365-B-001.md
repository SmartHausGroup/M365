# Lemma: Admin-only actions unreachable from User persona

-----------------------------------------
**Lemma ID:** LEM-M365-B-001-01

**Invariant:** INV-M365-B-001

**Derived from:**
- Section 2: Instruction Structure (\(\mathcal{A}_{\mathsf{Admin}}\), \(\mathcal{A}_{\mathsf{User}}\), disjoint)
- Section 4: Mutation vs Read Semantics (\(\mathcal{A}_{\mathrm{mut}} \subseteq \mathcal{A}_{\mathsf{Admin}}\))

**Statement:**

For every instruction \(x = (p, i, a, \mathit{params})\) with \(p = \mathsf{User}\): \(a \in \mathcal{A}_{\mathsf{User}}\), and therefore \(a \notin \mathcal{A}_{\mathsf{Admin}}\) and \(a \notin \mathcal{A}_{\mathrm{mut}}\). Admin-only and mutating actions are unreachable when persona is User.

**Assumptions:**

- \(\mathcal{A}_{\mathsf{Admin}} \cap \mathcal{A}_{\mathsf{User}} = \varnothing\) (Section 2).
- Syntactic validity: \((p = \mathsf{Admin} \wedge a \in \mathcal{A}_{\mathsf{Admin}}) \vee (p = \mathsf{User} \wedge a \in \mathcal{A}_{\mathsf{User}})\) for any instruction accepted for evaluation (Section 2; see also Section 6 for undefinedness when invalid).
- \(\mathcal{A}_{\mathrm{mut}} \subseteq \mathcal{A}_{\mathsf{Admin}}\) (Section 4).

**Proof outline:**

1. Consider any instruction \(x = (p, i, a, \mathit{params})\) with \(p = \mathsf{User}\) that is accepted for evaluation (so \(\mathrm{Eval}(x, \tau)\) is defined for some \(\tau\)).
2. By syntactic validity (Section 2), every accepted instruction satisfies \((p = \mathsf{Admin} \wedge a \in \mathcal{A}_{\mathsf{Admin}}) \vee (p = \mathsf{User} \wedge a \in \mathcal{A}_{\mathsf{User}})\).
3. Since \(p = \mathsf{User}\), the first disjunct is false; therefore the second must hold: \(a \in \mathcal{A}_{\mathsf{User}}\).
4. By disjointness, \(\mathcal{A}_{\mathsf{Admin}} \cap \mathcal{A}_{\mathsf{User}} = \varnothing\), so \(a \in \mathcal{A}_{\mathsf{User}}\) implies \(a \notin \mathcal{A}_{\mathsf{Admin}}\).
5. By Section 4, \(\mathcal{A}_{\mathrm{mut}} \subseteq \mathcal{A}_{\mathsf{Admin}}\), so \(a \notin \mathcal{A}_{\mathsf{Admin}}\) implies \(a \notin \mathcal{A}_{\mathrm{mut}}\).
6. Therefore for every instruction with persona User that is accepted for evaluation, \(a \in \mathcal{A}_{\mathsf{User}}\), \(a \notin \mathcal{A}_{\mathsf{Admin}}\), and \(a \notin \mathcal{A}_{\mathrm{mut}}\).

**Edge cases:**

- If \(p = \mathsf{User}\) and \(a \in \mathcal{A}_{\mathsf{Admin}}\), then the instruction is not syntactically valid; \(\mathrm{Eval}\) is undefined (Section 6), so the instruction is not "accepted" in the sense of producing a defined outcome. The invariant's pass condition applies to instructions that are evaluated; for those with \(p = \mathsf{User}\), the action is in \(\mathcal{A}_{\mathsf{User}}\) only.

**Conclusion:**

Syntactic validity and disjointness of action sets, together with \(\mathcal{A}_{\mathrm{mut}} \subseteq \mathcal{A}_{\mathsf{Admin}}\), imply that when persona is User the action is in \(\mathcal{A}_{\mathsf{User}}\) only and hence not in \(\mathcal{A}_{\mathsf{Admin}}\) or \(\mathcal{A}_{\mathrm{mut}}\). Hence INV-M365-B-001 holds.
