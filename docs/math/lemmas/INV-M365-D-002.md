# Lemma: Mutating actions are Admin-only

-----------------------------------------
**Lemma ID:** LEM-M365-D-002-01

**Invariant:** INV-M365-D-002

**Derived from:**
- Section 4: Mutation vs Read Semantics (mutating, persona and mutation)

**Statement:**

For every instruction \(x = (p, i, a, \mathit{params})\) with \(a \in \mathcal{A}_{\mathrm{mut}}\) that is accepted for evaluation, \(p = \mathsf{Admin}\). Equivalently: for every such instruction, persona is Admin; if persona is not Admin then the action is not in \(\mathcal{A}_{\mathrm{mut}}\).

**Assumptions:**

- Section 4: "Only Admin may perform mutating actions" and \(\mathcal{A}_{\mathrm{mut}} \subseteq \mathcal{A}_{\mathsf{Admin}}\).
- Section 2: syntactically valid instructions satisfy \((p = \mathsf{Admin} \wedge a \in \mathcal{A}_{\mathsf{Admin}}) \vee (p = \mathsf{User} \wedge a \in \mathcal{A}_{\mathsf{User}})\), with \(\mathcal{A}_{\mathsf{Admin}} \cap \mathcal{A}_{\mathsf{User}} = \varnothing\).

**Proof outline:**

1. By Section 4, \(\mathcal{A}_{\mathrm{mut}} \subseteq \mathcal{A}_{\mathsf{Admin}}\). So every \(a \in \mathcal{A}_{\mathrm{mut}}\) satisfies \(a \in \mathcal{A}_{\mathsf{Admin}}\).
2. By disjointness (Section 2), \(a \in \mathcal{A}_{\mathsf{Admin}}\) implies \(a \notin \mathcal{A}_{\mathsf{User}}\).
3. For any instruction accepted for evaluation with \(a \in \mathcal{A}_{\mathrm{mut}}\), we have \(a \in \mathcal{A}_{\mathsf{Admin}}\) (step 1). Syntactic validity requires either \((p = \mathsf{Admin} \wedge a \in \mathcal{A}_{\mathsf{Admin}})\) or \((p = \mathsf{User} \wedge a \in \mathcal{A}_{\mathsf{User}})\). The second disjunct is false because \(a \notin \mathcal{A}_{\mathsf{User}}\). Hence \(p = \mathsf{Admin}\).
4. Therefore for every instruction with \(a \in \mathcal{A}_{\mathrm{mut}}\) that is accepted for evaluation, \(p = \mathsf{Admin}\). So \(\mathrm{persona} \neq \mathsf{Admin}\) implies \(a \notin \mathcal{A}_{\mathrm{mut}}\) (contrapositive).

**Edge cases:**

- If \(p = \mathsf{User}\) and \(a \in \mathcal{A}_{\mathrm{mut}}\), then \(a \in \mathcal{A}_{\mathsf{Admin}}\), so the instruction is not syntactically valid; \(\mathrm{Eval}\) is undefined (Section 6). No such instruction receives a defined evaluation.

**Conclusion:**

The inclusion \(\mathcal{A}_{\mathrm{mut}} \subseteq \mathcal{A}_{\mathsf{Admin}}\) and the definition of syntactic validity imply that whenever the action is mutating and evaluation is defined, the persona is Admin. Hence INV-M365-D-002 holds.
